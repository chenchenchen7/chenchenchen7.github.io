Anaconda3+tensorflow2.0快速安装
发表于2020-08-19|更新于2020-08-25
|阅读量:

清华源下载最新版本Anaconda3

https://mirrors.tuna.tsinghua.edu.cn/anaconda/archive/?C=M&O=A

不用担心Anaconda自带的Python版本，反正最后自己要用conda管理新建一个新的环境，到时候会指定Python版本。

一路next下来，记得加入环境变量。最后打开Anaconda Prompt。最好管理员身份运行

创建新的环境并安装tf

conda create --name tf2.0.0 python=3.6

tf2.0.0就是新环境的名字，可以根据自己的需求更改，python选择3.6版本更稳妥，大部分tensorflow跟pytorch的版本都符合。

conda activate tf2.0.0

进入新环境，进行tf-gpu的安装。

conda install TensorFlow-gpu==2.0.0

conda会自动下载安装cuda和cudnn，所以不需要管。

jupyter notebook配置

在conda里安装了新环境，在jupyter里面是没有的，需要自己添加一下

首先进入到环境里

conda activate tf2.0.0

然后安装ipykernel

conda install ipykernel

写入jupyter的kernel

python -m ipykernel install --user --name 环境名称 --display-name "Python (环境名称)"

配置Pytorch排坑

CUDA Error： no kernel image is available for execution on device

显卡不行，计算能力太低。所以需要降低Pytorch的版本，1.1.0 1.2.0版本均可，只能先用低版本了。

但是降低版本，想要用GPU做计算，又必须让CUDA、Cudnn的版本对应。

这次改为Pytorch1.1.0版本，进入清华源、或Pytorch官网可以查看到对应的CUDA与Cudnn的版本：

https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/pytorch/win-64/

https://pytorch.org/get-started/previous-versions/

可以看到v1.1.0对应的CUDA为9.0/10.0，v1.2.0对应的CUDA为9.2/10.0
必须严格安装这个来安装CUDA版本，否者conda install的时候还会报错，找不到对应的包，HTTP请求失败。
如：CondaHTTPError: HTTP 503 SERVICE UNAVAILABLE:

从清华源看版本对应：
下面图中红色框内，可以看到1.1.0可以选择python的版本为3.5-3.7，但是cuda90对应cudnn7.1，cuda10.0对应cudnn7.1
1.2.0可以选择python的版本为3.5-3.7，但是cuda9.2对应cudnn7.1，cuda10.0对应cudnn7.1

由于之前没有仔细查看，所以安装了CUDA9.0，但是从图里看，选择cuda10.0与cudnn7.1，能涵盖更多版本的pytorch。
所以推荐第一开始配置，就选择CUDA10.0，Cudnn7.1。

Anaconda删除、重命名环境

创建新环境

conda create -n rcnn python=3.6

删除环境

conda remove -n rcnn --all

重命名环境

conda 其实没有重命名指令，实现重命名是通过 clone 完成的，分两步：

  • 先 clone 一份 new name 的环境
  • 删除 old name 的环境

第1步
conda create -n tf --clone rcnn

第2步
conda remove -n rcnn --all
