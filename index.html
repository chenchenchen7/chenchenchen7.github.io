PINNs : Physics Informed Neural Networks

文章分两部分

  1. Data-driven solutions
  2. Data-driven discovery

两大部分里又分别讲了连续模型和离散模型。
以下面的图为例，为求解Burgers方程的连续与离散解法。

对于连续模型来说：

  • inference指，用已知的边界及边界值(边界条件)**和方程参数**，可以学习推理出内部所有的值；
  • identification指，用已知的边界及内部值，来学习到方程的参数，如lambda1, lambda2。

对于离散时间模型来说：

  • inference指，用已知的某一时刻的数据值和方程参数，可以学习推理出下一时刻值；
  • identification指，用已知的已经多时刻的值，来学习到方程的参数，如lambda1, lambda2。

所以下面两个可以归类于Data-driven solutions，因为是去解方程，根据已知的边界和已知方程及参数，去得到区域内所有的值。

那么余下的两个identification就是discovery，根据已知的大量的点值，求解得到方程参数。

Data-driven solutions of partial differential equations

就是在讲怎么解 PDEs（Partial Differential Equations）

continuous time inference (Burgers)

用已知的边界及边界值和方程参数，可以学习推理出内部所有的值

discrete time inference (Burgers)

用已知的某一时刻的数据值和方程参数，可以学习推理出下一时刻值；

Data-driven discovery of partial differential equations

就是在讲怎样解PDE的反问题 (带参数的PDE，参数需要在解的过程中求解出来)

continuous time identification (Burgers)

用已知的边界及内部值，来学习到方程的参数，如lambda1, lambda2，图的最下方为模型识别出的lambda1和lambda2，可以看出与原方程误差很小。

discrete time identification (Burgers)

用已知的已经多时刻的值，来学习到方程的参数，如lambda1, lambda2。

1.清华源下载最新版本Anaconda3

https://mirrors.tuna.tsinghua.edu.cn/anaconda/archive/?C=M&O=A

不用担心Anaconda自带的Python版本，反正最后自己要用conda管理新建一个新的环境，到时候会指定Python版本。

一路next下来，记得加入环境变量。最后打开Anaconda Prompt。最好管理员身份运行

2.创建新的环境并安装tf

conda create --name tf2.0.0 python=3.6

tf2.0.0就是新环境的名字，可以根据自己的需求更改，python选择3.6版本更稳妥，大部分tensorflow跟pytorch的版本都符合。

conda activate tf2.0.0

进入新环境，进行tf-gpu的安装。

conda install TensorFlow-gpu==2.0.0

conda会自动下载安装cuda和cudnn，所以不需要管。

Java项目

我是一个Java的新手，差不多学到多线程，想问下🐏哥，在GitHub下载你推荐的Java练习项目该怎么入手。

———————看readme文档，所提到的技术点各个击破———————

—–读好readme，或者项目自带的文档说明，基本就问题不大了，前提是基础和框架部分要熟悉鸭—–

  1. Java

    项目简介：这是一个基于Java的数据结构与算法的实现项目。里面包含了几乎常用所有数据结构的实现，以及诸多算法题和LeetCode习题的Java实现。主要包括：

    • 加解密算法
    • 进制转换
    • 各种数据结构
    • 分治
    • 动态规划
    • 数学类问题
    • 搜索问题
    • 排序问题
    • LeetCode习题
    • 其他杂项算法题等

    基于Java语言的数据结构和算法题练手必备！

    项目源码：https://github.com/TheAlgorithms/Java

  2. eladmin

    项目简介：eladmin是一个基于 Spring Boot + Vue的前后端分离的后台管理系统，项目采用分模块开发方式， 权限控制采用 RBAC，支持数据字典与数据权限管理，支持一键生成前后端代码，支持动态路由，对于初学者还是比较友好的。

    • Spring Boot 2.x

    • JPA

    • Spring Security

    • Redis

    • Vue

      一套典型的后台管理系统，用的也是一套比较典型的Java后端开发技术，可以帮我们串联起很多后端开发的知识。

    项目源码：https://github.com/elunez/eladmin

  3. jodd

    项目简介：Jodd = tools + ioc + mvc + db + aop + tx + json + html < 1.7 Mb

    没错，Jodd是一个非常易用和好上手的开源Java微框架，里面包含了一系列平时经常会用到的一些核心程序库、工具类/方法、实用程序/框架等等。有了它，开发人员做起事来会变得非常简易和优雅，引入jodd就能帮我们快速实现某些功能。点赞！

    jodd主要包含：

    • Jodd Libraries（程序库）：Json、Email、HTTP、Jerry …
    • Micro-frameworks（微框架）：Madvoc、Petite、Proxetta、DbOom …
    • Jodd Utilities（实用工具）：BeanUtil、Props、Decora、Cli、Ref …

    一方面当我们想要用Java快速实现一些功能需求的时候，引入jodd就可以帮我们完成很多事情，很多工具和代码都开箱即用，简便高效；另外一方面研究其源码可以帮助我们打开技术视野。

    项目源码：https://github.com/oblac/jodd

  4. SnowJena

    项目简介：SnowJena是一个基于令牌桶算法实现的分布式无锁限流框架，支持熔断降级，支持动态配置规则，支持可视化监控，开箱即用。可用于Java后端项目常见的本地限流和分布式限流的场景。

    jodd主要包含：

    • 支持本地限流
    • 支持分布式限流
    • 支持黑白名单
    • 支持可视化监控等。

    一是帮助我们学习和实践限流这一常见的技术方案和实现原理，二是帮助我们学习和理解常见的设计模式。

    项目源码：https://github.com/ystcode/SnowJena

  5. seata

    项目简介：Seata 是一款阿里巴巴开源的分布式事务解决方案，致力于在微服务架构下提供高性能和简单易用的分布式事务服务。

    jodd主要包含：

    • 支持常见主流的微服务框架
    • 支持AT模式
    • 支持TCC模式
    • 支持SAGA模式
    • 高可用和强大的横向扩展能力

    分布式事务问题几乎是当下后端开发和架构领域最棘手和最有含金量的问题之一，多学点总是好的。

    项目源码：https://github.com/seata/seata
